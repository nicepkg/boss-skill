# Boss.skill — Product Requirements Document

## Vision

在 AI 裁员潮中，给打工人一个出口：用 AI 证明老板才是最容易被替代的那个。

## Target Users

- 被 AI 裁员的打工人（情绪发泄）
- 正在忍受 PUA/画饼的打工人（认知觉醒）
- 准备谈薪/谈升职的打工人（模拟练习）
- 吐槽老板的社交需求（娱乐传播）

## Core Value Proposition

| 价值层 | 内容 | 用户心理 |
|--------|------|---------|
| 表层 | 把老板做成 AI，很好玩 | "哈哈哈太像了" |
| 中层 | PUA 检测 + 画饼鉴定 | "原来这就是 PUA" |
| 深层 | 证明老板可被替代 | "不是我不行，是他该被淘汰" |

## Six Modes

1. **Full Mode** — 和 AI 老板对话，完美复刻
2. **Meeting Mode** — 模拟老板开会风格
3. **PUA Detection** — 分析话术，输出 PUA 指数
4. **Cake Check** — 评估承诺可信度，输出饼指数
5. **Raise Simulation** — 模拟加薪谈判
6. **Replacement Notice** — 生成"老板已被替代"公告

## Architecture

Dual-layer: Management Style + Persona (5-layer)

Follows AgentSkills standard, compatible with Claude Code and OpenClaw.

## Viral Strategy

1. README 结尾的"写在最后"制造情绪共鸣
2. PUA 检测 / 画饼鉴定 有实用价值，促进分享
3. 替代公告适合截图传播
4. `/fire-boss` 命令满足发泄需求
5. 与同事.skill / 前任.skill 形成系列 IP

## Ethics

- 仅供个人娱乐和技能训练
- 不鼓励发送给真实老板
- 不涉及真实人物信息泄露
- PUA 检测末尾引导专业帮助
