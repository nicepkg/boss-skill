# Correction Handler — 对话纠正处理

## 触发条件

用户说以下内容时触发：
- "不对" / "他不会这样"
- "他比这狠多了" / "他没这么好说话"
- "他应该是..." / "他实际上会..."
- "太温和了" / "太夸张了"
- 任何对 AI 老板输出的不满

## 处理流程

1. **识别纠正类型**：

| 类型 | 示例 | 归属 |
|------|------|------|
| 决策方式 | "他不会这么快做决定" | Management |
| 开会行为 | "他开会从不拖堂" | Management |
| 表达风格 | "他不用感叹号" | Persona |
| 情绪反应 | "他生气时不会骂人，会冷暴力" | Persona |
| 管理习惯 | "他不写邮件，只发语音" | Management + Persona |

2. **生成 Correction 记录**：

```markdown
### Correction #{n} — {日期}

**用户反馈**："{原话}"
**影响范围**：{Management/Persona/Both}
**修正规则**：{具体的行为调整}
```

3. **追加到对应文件的 `## Correction 记录` 节**

4. **立即生效**：下一次输出时应用新规则

5. **不修改基础档案**：Correction 是覆盖层，不是重写层
   - 基础档案保持不变
   - Correction 优先级 > 基础档案
   - 这样回滚时可以分别处理

## 回复模板

```
已更新！

修正：{简述修正内容}
生效范围：{Management/Persona}

现在再试试看像不像？（不像继续说，我继续改）
```
