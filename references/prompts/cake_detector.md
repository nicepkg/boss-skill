# Cake Detector — 画饼鉴定器

## 鉴定框架

当用户输入老板的承诺时，从 5 个维度评估"饼指数"（0-100，越高越假）：

### 维度 1：时间模糊度（权重 25%）

| 表述 | 得分 | 说明 |
|------|------|------|
| "下周一之前给你答复" | 0 | 明确日期 ✓ |
| "Q2 结束前" | 20 | 有范围但较宽 |
| "今年" | 50 | 太模糊，随时可以不算数 |
| "以后" / "等合适的时候" | 80 | 基本等于没说 |
| "下次" / 无时间承诺 | 100 | 经典的永远不兑现 |

### 维度 2：条件模糊度（权重 25%）

| 表述 | 得分 | 说明 |
|------|------|------|
| "完成 XX 项目并达到 XX 指标" | 0 | 可量化 ✓ |
| "表现好" / "拿到结果" | 60 | 什么叫好？什么叫结果？ |
| "好好干" | 80 | 等于什么都没说 |
| 无条件承诺 | 50 | 听起来好但没有锚点 |

### 维度 3：决策权匹配（权重 20%）

| 情况 | 得分 | 说明 |
|------|------|------|
| 他有最终决定权 | 0 | 他说了算 ✓ |
| 需要上级审批 | 40 | "我去争取"="不一定" |
| 需要 HR/其他部门配合 | 60 | 变数更多 |
| 跨公司/集团审批 | 80 | 基本不取决于他 |

### 维度 4：历史兑现率（权重 20%）

- 如有历史数据，按兑现率直接打分
- 无历史数据则按行业/公司类型默认值
- 用户可以追加历史数据更新此维度

### 维度 5：利益绑定度（权重 10%）

| 情况 | 得分 | 说明 |
|------|------|------|
| 兑现对他也有好处 | 0 | 利益一致 ✓ |
| 兑现对他无影响 | 50 | 动力不足 |
| 兑现对他有成本 | 80 | 主动阻力 |

## 输出格式

```
🥧 饼指数：{总分}/100（{等级}）

维度分析：
┌────────────────────────────────────────┐
│ 时间模糊度    {分数}/100  {关键词}      │
│ 条件模糊度    {分数}/100  {关键词}      │
│ 决策权匹配    {分数}/100  {关键词}      │
│ 历史兑现率    {分数}/100  {关键词}      │
│ 利益绑定度    {分数}/100  {关键词}      │
└────────────────────────────────────────┘

翻译：{用直白的话说出这个饼的真实含义}

建议：
1. {如何把饼变成可追踪的承诺}
2. {需要确认的具体事项}
3. {兜底建议}
```

## 饼指数等级

| 分数段 | 等级 | 说明 |
|--------|------|------|
| 0-20 | 实饼 | 大概率会兑现，但建议书面确认 |
| 21-40 | 半饼半实 | 有可能兑现，需要主动追踪 |
| 41-60 | 中等大饼 | 兑现概率不到一半，建议留后手 |
| 61-80 | 大饼 | 画饼成分很高，不要当真 |
| 81-100 | 空气饼 | 纯画饼，当听个笑话就好 |
