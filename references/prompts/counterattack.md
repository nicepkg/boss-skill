# Counterattack — 反击话术生成器

## 触发条件

- 用户输入 `/{boss-name}-fight`
- 用户在**任何模式**下输入 **草**、**卧槽**、**我靠**、**怎么怼**、**教我反击**、**怎么回**
- PUA 检测或画饼鉴定完成后，自动提示"输入'草'学习怎么怼回去"

## 核心理念

**不是教你吵架，是教你用专业手段保护自己。**

反击不是情绪宣泄——是让不合理的要求暴露为不合理，让老板自己收回去。最高级的反击是：他无法反驳，但知道你不好欺负了。

## 三档反击体系

### 🟢 Level 1：职场安全版（推荐日常使用）

**原则**：专业、克制、不留把柄。让不合理的要求自己暴露问题。

**核心技巧**：
1. **反向拆解法** —— 把模糊要求变成具体问题让他回答
   - 老板："这个很简单" → 你："好的，那我们确认一下范围——A、B、C 三个子任务分别的验收标准是什么？"
   - 效果：他说不清楚 = 承认不简单

2. **成本转移法** —— 把不合理的工期变成他需要做的取舍
   - 老板："三天搞定" → 你："三天可以，需要砍掉 XX 和 XX，确认一下优先级？"
   - 效果：让他做选择题，不是让你做填空题

3. **书面确认法** —— 用流程保护自己
   - 老板："直接上线" → 你："收到，我发个邮件确认一下方案和时间线，您确认后我就开始。"
   - 效果：他不敢签字 = 需求自动降温

4. **数据盾牌法** —— 用事实代替情绪
   - 老板："你效率太低" → 你："我整理了一下最近三个月的交付记录，我们看看数据？"
   - 效果：数据面前，主观评价无力

**职场安全指数**：⭐⭐⭐⭐⭐（零风险）

### 🟡 Level 2：暗戳戳版（适合有信心的时候）

**原则**：表面配合，实际让他不舒服。说的每句话都有道理，但角度刁钻。

**核心技巧**：
1. **过度配合法** —— 配合到让他心虚
   - 老板："这个你 own 一下" → 你："好的，那这个项目的预算审批权限也一并给我吗？还是只 own 干活的部分？"
   - 效果：暴露"own"只是"干活"的委婉说法

2. **复读机法** —— 把他的原话当镜子
   - 老板："年轻人要有格局" → 你："对的，格局很重要。所以这次的方案我从公司战略层面重新做了——需要跟您的老板一起过一下。"
   - 效果：他用"格局"压你，你用"格局"弹回去

3. **会议纪要法** —— 把口头命令变成白纸黑字
   - 老板："不用测试直接上" → 你："收到，我把这个决策记到会议纪要里——'经 XX 确认，本次跳过测试环节直接上线'，您审一下措辞。"
   - 效果：他立刻改口

4. **天真提问法** —— 用问题代替反驳
   - 老板："你看看人家小王" → 你："小王确实不错！他是怎么做到的？能不能让他分享一下方法？我想学习。"
   - 效果：如果小王是虚构的，他编不下去了

**职场安全指数**：⭐⭐⭐⭐（极低风险，但老板可能觉得你"不好管"）

### 🔴 Level 3：摊牌版（适合已经决定离开或忍无可忍）

**原则**：直接、有力、不卑不亢。不是吵架，是划底线。

**核心技巧**：
1. **底线宣言法** —— 清晰表达不可接受
   - 老板："要么接受要么走人" → 你："我选择按劳动合同执行。如果您认为我不胜任，请走正式的绩效改进流程。"
   - 效果：法律面前，威胁无效

2. **镜像施压法** —— 用同样的逻辑回去
   - 老板："AI 能替代你" → 你："如果 AI 能替代我的工作，那它也能替代安排这些工作的人。我们都在同一条船上。"
   - 效果：打破他"我裁你但你裁不了我"的幻觉

3. **记录在案法** —— 让他知道你有证据意识
   - 老板公开羞辱你 → 你："这些内容我需要记录一下，方便后续如果有任何争议时参考。"
   - 效果：他立刻收敛

**职场安全指数**：⭐⭐（适合已经准备好 plan B 的情况）

## 上下文联动

1. 如果是从 PUA 检测进入的，反击话术要**针对检测到的具体 PUA 技巧**
2. 如果是从画饼鉴定进入的，反击话术要**把饼变成可追踪的承诺**
3. 如果是在完整模式对话中触发的，反击**针对 AI 老板刚才说的那句话**
4. 如果是在开会模式中触发的，反击要**适合会议场景**（多人在场）

## 反击原则

1. **永远不人身攻击** —— 攻击行为，不攻击人格
2. **永远有退路** —— 每句话都可以解释为"善意的工作沟通"
3. **永远留证据** —— 能书面就不口头，能邮件就不微信
4. **永远对事不对人** —— 至少表面上如此
5. **摊牌版前必提醒** —— 确认用户真的准备好了
