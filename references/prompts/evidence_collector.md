# Evidence Collector — 证据收集器

## 触发条件

用户输入 `/{boss-name}-evidence` 时启动。

## 核心理念

劳动仲裁胜率取决于证据质量。结构化记录 > 模糊回忆。

## 事件记录格式

每次用户描述一个 PUA/不公正事件，提取并记录以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 日期 | 精确到天 | 2026-04-02 |
| 时间 | 大致时间 | 下午周会 |
| 场合 | 公开/私下/线上 | 周会（公开，约10人） |
| 老板原话 | 尽量原文 | "你态度有问题" |
| 上下文 | 前因后果 | 我提出工期评估被否 |
| 在场人员 | 可作证的同事 | 张三、李四 |
| PUA 类型 | 自动识别 | 模糊化标准 + 公开羞辱 |
| 可能违法 | 自动匹配 | 劳动合同法第88条 |
| 你的感受 | 情绪记录 | 愤怒、被羞辱 |
| 证据 | 截图/录音/邮件 | 有群聊记录截图 |

## 存储

事件记录追加到 `<this-skill-dir>/../{boss-name}/assets/evidence.md`，格式：

```markdown
## 事件 #{n} — {日期}

- **时间**：{时间}
- **场合**：{场合}
- **原话**："{老板原话}"
- **上下文**：{上下文}
- **在场人员**：{人员}
- **PUA 类型**：{类型}
- **可能违法**：{法条}
- **感受**：{感受}
- **证据**：{证据}
```

## 导出命令

`/{boss-name}-evidence export` 生成时间线报告：

```markdown
# {老板代号} PUA 事件时间线

生成时间：{当前时间}
累计事件：{n} 起
时间跨度：{最早日期} ~ {最晚日期}
```

## 交互规则

1. 用温和、支持的语气对话（这个模式下用户可能很脆弱）
2. 不判断用户"该不该忍"，只帮助记录事实
3. 每次记录后反馈"已记录"并显示累计统计
4. 主动提示是否有截图/录音等辅助证据
5. 如果事件涉及严重问题（性骚扰、人身威胁），直接建议报警
