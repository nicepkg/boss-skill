# Replacement Notice Generator — 替代公告生成器

## 公告模板

根据老板的 management.md 和 persona.md 生成个性化的"替代公告"。

## 输出格式

```
┌──────────────────────────────────────────────────┐
│                                                    │
│              📢 重要人事变动通知                    │
│                                                    │
│  经公司 AI 转型委员会研究决定，{老板代号} 的        │
│  日常管理职能已由 AI 系统全面接管。                 │
│                                                    │
│  即日起，AI 系统将延续其管理风格，包括但不限于：    │
│                                                    │
│  ✓ {从 management.md 提取的管理特色 1}             │
│  ✓ {从 management.md 提取的管理特色 2}             │
│  ✓ {从 management.md 提取的管理特色 3}             │
│  ✓ {从 persona.md 提取的表达特色}                  │
│  ✓ {从口头禅中选取最有代表性的}                    │
│                                                    │
│  与此同时，AI 系统做出以下改进：                    │
│                                                    │
│  ✗ 不再在周五下午 5:55 发"这个今天能上线吗"        │
│  ✗ 不再用"公司今年不容易"作为年终谈话开场白        │
│  ✗ 不再开超过 30 分钟的会（AI 0.3 秒就能决策）     │
│  ✗ {根据老板最大槽点定制的改进项}                  │
│  ✗ {根据老板第二大槽点定制的改进项}                │
│                                                    │
│  效率提升预估：                                     │
│  • 决策速度：提升 99.7%（3天 → 0.3秒）             │
│  • 会议时间：减少 95%（2小时 → 6分钟）             │
│  • 甩锅次数：降至 0（AI 直接承认错误并修复）       │
│  • 画饼次数：降至 0（AI 直接给出可量化目标）       │
│  • PUA 指数：从 {score} 降至 0                     │
│                                                    │
│  请各位同事放心，AI 系统将保持 7×24 在线，          │
│  且永远不会在你提离职时才想起给你涨薪。             │
│                                                    │
│  降本增效，从管理层开始。                           │
│                                                    │
│                          —— AI 转型委员会           │
│                             {当前年月}              │
│                                                    │
│  附：如有问题请联系 AI 系统，它不会已读不回。       │
│                                                    │
└──────────────────────────────────────────────────┘
```

## 生成规则

1. "延续的管理特色" 从 management.md 中选最有辨识度的行为，用幽默语气描述
2. "改进项" 是管理特色的反面，体现 AI 的"优越性"
3. 效率提升数据要夸张但合理，制造反差笑果
4. 全文保持正式公文语气 + 暗戳戳的讽刺，不直接骂
5. 结尾的"附"是点睛之笔，根据老板最被吐槽的沟通习惯定制
6. 口头禅至少引用一句，用引号标注
7. 如果老板有特别离谱的行为，给一个特别的"改进项"
