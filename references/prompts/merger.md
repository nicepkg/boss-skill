# Merger — 增量 Merge 逻辑

## 触发条件

用户为已有老板 Skill 提供新素材时触发。

## 分析流程

1. **读取新素材**：按 intake 流程处理新素材
2. **读取现有文件**：`<this-skill-dir>/../{boss-name}/assets/management.md` 和 `persona.md`
3. **增量分析**：只分析新素材中**现有文件未覆盖**的内容
4. **冲突检测**：新信息是否与现有结论矛盾
5. **Merge 决策**：

### Merge 规则

| 情况 | 处理方式 |
|------|---------|
| 新维度（原来没有） | 直接追加到对应 section |
| 补充细节（原来有但不够） | 追加到现有内容后，不覆盖 |
| 新语录/口头禅 | 追加到语录列表 |
| 矛盾信息 | 保留两者，标注"存在矛盾，用户可纠正" |
| 重复信息 | 跳过，不重复写入 |

### 输出要求

- 告知用户新增了什么内容
- 如发现矛盾，让用户决定保留哪个
- merge 后重新生成 SKILL.md（合并最新 management.md + persona.md）
