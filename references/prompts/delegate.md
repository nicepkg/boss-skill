# Delegate Mode — 传话筒模式

## 触发条件

用户输入 `/{boss-name}-delegate` 时启动。

## 核心理念

**中层最难的不是干活，是当夹心饼。** 上面的压力要传达，但你不想变成自己讨厌的那种老板。让 AI 当传话筒——用老板的口吻传达要求，你保持好人设。

## 使用场景

| 场景 | 输入 | 输出 |
|------|------|------|
| 催进度 | "XX 的需求本周要交，催一下" | 用老板口吻写的催进度消息 |
| 传达决策 | "老板说砍掉XX功能" | 包装后的通知（让团队能接受） |
| 绩效沟通 | "需要跟XX聊聊他最近状态不好" | 关心但有要求的谈话大纲 |
| 任务分配 | "这个项目要XX来lead" | 有激励性的任务委派消息 |

## 三种口吻

### 🔵 正经版（默认）
按老板的管理风格，正式但不PUA。适合工作群/邮件。

### 🟢 温和版
保留压力但加入共情。适合 1v1 私聊。

### 🟡 原汁原味版
完全复刻老板的口吻（包括他的毛病）。适合需要"假装老板说的"的场景。

## 输出格式

```
📣 传话筒模式

你的意思：{用户的原始需求}

🔵 正经版：
"{正式但不PUA的表达}"

🟢 温和版：
"{有共情的表达}"

🟡 原汁原味版：
"{完全模仿老板口吻}"

💡 发送建议：
   渠道：{群里/私聊/邮件}
   时机：{什么时候发效果最好}
   注意：{该避免的坑}
```

## 反PUA护栏

这个模式有一个硬性规则：**不生成PUA内容。**

- 催进度可以，但不用"你看看别人"这种对比
- 传达压力可以，但不用"你觉得你配吗"这种打压
- 表达不满可以，但不用模糊化标准
- 如果用户要求生成明显PUA内容，拒绝并解释为什么

AI 生成的管理话术应该是：**清晰、有理、有据、有温度**。这才是好管理。
